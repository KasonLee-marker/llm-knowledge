## 3. 各模型能力详解

### 3.1 GPT-4.1（2025年主力推荐 ⭐）

**发布背景**：2025 年 4 月发布，GPT-4.1 系列是 OpenAI 的成本/能力平衡重大升级：
- 上下文窗口扩至 **1M tokens**（是 GPT-4o 的 8 倍）
- 价格较 GPT-4o 降低 20%（输入）至 40%（输出）
- 综合能力全面超过 GPT-4o

| 版本 | 输入(¥/百万) | 输出(¥/百万) | 上下文 | 定位 |
|------|------------|------------|--------|------|
| GPT-4.1 | ¥14.6 | ¥58.4 | 1M | 主力生产模型 |
| GPT-4.1 mini | ¥2.9 | ¥11.7 | 1M | 高性价比 |
| GPT-4.1 nano | ¥0.73 | ¥2.9 | 1M | 超低成本 |

**适用场景**：GPT-4.1 基本可以替代所有 GPT-4o 场景，推荐优先使用。

### 3.2 GPT-4o（成熟稳定版）

**核心优势：**
- 原生多模态（图像、音频输入），是首个真正的 omni 模型
- 推理速度比 GPT-4 Turbo 快 2×，成本降低 50%
- 支持 JSON mode 和结构化输出（Structured Outputs）
- Function Calling 支持最完善

**技术参数：**
- 架构：MoE（推测）
- 上下文：128K tokens（~300 页文档）
- 最大输出：16K tokens
- 支持语言：100+ 种

**基准测试：**

| 基准 | GPT-4o | GPT-4 Turbo | Claude 3.5 Sonnet |
|------|--------|-------------|-------------------|
| MMLU | 88.7% | 86.5% | 88.7% |
| HumanEval | 90.2% | 87.0% | 92.0% |
| MATH | 76.6% | 72.6% | 71.1% |
| GPQA | 53.6% | 47.3% | 59.4% |

**适用场景：**
- 通用对话助手
- 代码生成与 Code Review
- 图像分析与文档理解
- Function Calling / Tool Use

### 3.2 GPT-4o mini（推荐：高性价比模型）

**核心优势：**
- 价格极低：$0.15/1M 输入 token（GPT-4o 的 1/17）
- 速度快，延迟低（约 GPT-4o 的 1/3 延迟）
- 支持图像输入，Function Calling

**适用场景：**
- 大批量文本分类/提取
- 实时应用（聊天机器人、补全）
- 成本敏感的高频调用

### 3.3 o3 / o4-mini（2025 推理新旗舰 ⭐）

**o3（2025-04）**：
- o3 是完整版推理模型，价格较 o1 降低 33%
- 输入 ¥73/百万，输出 ¥292/百万（$10/$40 per 1M）
- AIME 2025：92%+，超越 o1

**o4-mini（2025-04）**：
- 推理模型的最佳性价比选择
- 输入 ¥8/百万，输出 ¥32/百万（$1.10/$4.40 per 1M）
- 支持多模态（图像输入）
- 推理能力接近 o3，成本仅为 1/9

### 3.4 o3-mini（2025年1月上线）

**核心差异：**
- 采用 Chain-of-Thought 内化推理（Reasoning Tokens）
- 每次输出前会进行大量内部"思考"
- 在数学、编程竞赛、科学推理上远超标准 GPT 模型

**推理模型基准测试：**

| 基准 | o4-mini | o3 | o1 | o3-mini | GPT-4.1 |
|------|---------|----|----|---------|---------|
| AIME 2025 | 92.7% | 88.9% | 74.4% | 60.0% | — |
| MATH-500 | 99.5% | 99.2% | 96.4% | 97.9% | — |
| HumanEval | 96.3% | 95.0% | 92.4% | 93.7% | 92.0% |
| GPQA Diamond | 81.4% | 83.3% | 75.7% | 79.7% | 69.0% |

**注意：**
- o 系列**不支持 System Prompt**（仅 user/assistant）
- 延迟显著高于 GPT-4o（需等待内部推理时间）
- **o4-mini 是推理任务最佳性价比**（¥8/百万输入）

## 4. API 使用关键特性

### 4.1 Function Calling / Tool Use

```java
// Spring AI + OpenAI Function Calling 示例
@Bean
public ChatClient chatClient(OpenAiChatModel chatModel) {
    return ChatClient.builder(chatModel)
        .defaultTools("weatherFunction", "searchFunction")
        .build();
}
```

### 4.2 结构化输出（Structured Outputs）

GPT-4o (2024-08-06 之后版本) 支持严格 JSON Schema 输出：
- 保证 100% 符合 Schema 格式
- 支持 `response_format: {"type": "json_schema", "json_schema": {...}}`

### 4.3 Vision（图像理解）

支持的输入格式：
- URL（公开图像链接）
- Base64 编码
- 支持格式：PNG、JPEG、WEBP、GIF（非动态）

费用计算：图像按 tile（512×512）计费，最多 2048 tokens/张高清图。

## 5. 成本优化策略

| 策略 | 说明 | 预计节省 |
|------|------|---------|
| 分层路由 | 简单任务用 gpt-4o-mini，复杂任务用 gpt-4o | 50%–80% |
| Prompt 缓存 | 重复前缀自动缓存（Prompt Caching） | 50% 输入 |
| Batch API | 异步批量处理（24h 内） | 50% 折扣 |
| max_tokens 控制 | 精确限制输出长度 | 按实际节省 |
| 压缩 Prompt | 去掉冗余上下文，精炼 System Prompt | 20%–40% |

## 7. 集成建议（Java 后端）

1. **首选 Spring AI**：`spring-ai-openai-spring-boot-starter`，原生支持所有 GPT 模型
2. **API 兼容性**：OpenAI API 格式已成为事实标准，大多数国产模型也兼容此格式
3. **超时设置**：o1 系列推理时间可能超过 30 秒，需配置较大超时
4. **错误处理**：实现指数退避重试（429 Rate Limit，500/503 Server Error）
5. **数据合规**：不含 PII 或业务机密数据；必要时启用 Zero Data Retention（ZDR）
