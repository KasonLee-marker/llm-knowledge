## 3. 各模型能力详解

### 3.1 GPT-5.4（2026年旗舰 ⭐）

**发布背景**：2026 年 3 月发布，GPT-5.4 是 OpenAI 当前最强模型：
- 上下文窗口扩至 **1M tokens**
- 支持"Thinking"深度推理模式
- 多模态能力全面升级（图像/音频/视频）
- 更强的 Agent 和工具调用能力

| 版本 | 输入(¥/百万) | 输出(¥/百万) | 上下文 | 定位 |
|------|------------|------------|--------|------|
| GPT-5.4 (≤272K) | ¥18.3 | ¥109.5 | 1M | 标准旗舰 |
| GPT-5.4 (>272K) | ¥36.5 | ¥164 | 1M | 长上下文 |
| GPT-5.4 Pro (≤272K) | ¥219 | ¥1314 | 1M | 顶级性能 |
| GPT-5.4 Pro (>272K) | ¥438 | ¥1971 | 1M | 顶级长上下文 |

**适用场景**：复杂推理、长文档分析、多模态任务、Agent 应用

### 3.2 GPT-5.2 / GPT-5.1（主力生产模型 ⭐）

**核心优势：**
- GPT-5.2 在推理和代码能力上较 GPT-5 有显著提升
- 400K 上下文窗口，适合大多数生产场景
- 价格相对 GPT-5.4 更亲民
- 支持缓存输入（90% 折扣）

**技术参数：**
- 架构：MoE（混合专家）
- 上下文：400K tokens
- 最大输出：128K tokens
- 支持语言：100+ 种

**基准测试：**

| 基准 | GPT-5.2 | GPT-5.1 | GPT-5 | GPT-4.1 |

| HumanEval | 93.5% | 92.0% | 90.5% | 92.0% |

**适用场景：**
- 通用对话助手
- 代码生成与 Code Review

- Function Calling / Tool Use

## 4. API 使用关键特性

### 4.1 Function Calling / Tool Use

```java
// Spring AI + OpenAI Function Calling 示例
@Bean
public ChatClient chatClient(OpenAiChatModel chatModel) {
    return ChatClient.builder(chatModel)
        .defaultTools("weatherFunction", "searchFunction")
        .build();
}
```

### 4.2 结构化输出（Structured Outputs）

GPT-5 系列支持严格 JSON Schema 输出：
- 保证 100% 符合 Schema 格式
- 支持 `response_format: {"type": "json_schema", "json_schema": {...}}`

### 4.3 缓存输入（Prompt Caching）

GPT-5 系列支持自动缓存：
- 缓存命中价格仅为原价的 10%
- 自动识别重复前缀
- 大幅降低长上下文成本

### 4.4 Vision（图像理解）

支持的输入格式：
- URL（公开图像链接）
- Base64 编码
- 支持格式：PNG、JPEG、WEBP、GIF（非动态）

## 5. 成本优化策略

| 策略 | 说明 | 预计节省 |
|------|------|---------|
| 分层路由 | 简单任务用 GPT-5-nano，复杂任务用 GPT-5.2 | 70%–90% |
| Prompt 缓存 | 重复前缀自动缓存 | 90% 输入 |
| Batch API | 异步批量处理（24h 内） | 50% 折扣 |
| max_tokens 控制 | 精确限制输出长度 | 按实际节省 |
| 压缩 Prompt | 去掉冗余上下文，精炼 System Prompt | 20%–40% |

## 7. 集成建议（Java 后端）

1. **首选 Spring AI**：`spring-ai-openai-spring-boot-starter`，原生支持所有 GPT 模型
2. **API 兼容性**：OpenAI API 格式已成为事实标准
3. **超时设置**：o 系列推理时间可能超过 30 秒，需配置较大超时
4. **错误处理**：实现指数退避重试（429 Rate Limit，500/503 Server Error）
5. **数据合规**：不含 PII 或业务机密数据；必要时启用 Zero Data Retention（ZDR）
